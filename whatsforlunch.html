What's For Lunch

A website to vote on where to eat

Situation

    There have been countless times where nobody can decide where to eat or they nobody wants to suggest something.

Task

    To develop a voting app centered around voting for places to eat. The website needed user sign in ability, input fields for restaurants, and the ability to vote. Restaurants that have the top vote will receive a golden border to indicate they are leading. Users can vote for multiple restaurants but can only vote for that restaurant once.

Action

    I owned every step of the product from building the front-end in React and connecting it to the backend in Firebase.

Results

    The total development time of the project was about two months. The application allows for users to sign in and vote on where to eat for lunch. As well as show the current restaurant leader.

    Next steps would be to allow for multiple users to create there own instances of the voting app. At the moment, there can only be one instance of the application running per day. The application resets at midnight.

Technologies used

    HTML5 | CSS3 | JavaScript | ReactJS | Firebase | Github
